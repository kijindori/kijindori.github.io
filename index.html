플랫포머 게임

[개인 프로젝트]

소스코드

    개발 환경

    C++ / Win32 API Visual Studio 2022

    TCP/IP 소켓 프로그래밍을 통해서 클라이언트와 서버가 통신하여 플레이어의 위치를 여러 클라이언트 간에 동기화하도록 했습니다.

    이 과정에서 멀티 스레드 환경에서 임계 영역을 통해 동기화를 했고, 윈도우 OS 가 제공하는 비동기 입출력 처리 API를 활용했습니다.

    충돌 처리, 키보드 입력 처리, 플레이어 등 기능별로 클래스를 만들었고, 간결한 구조로 유지보수가 용이하게 하고자 노력했습니다.

콘텐츠 관리 크롬 확장 프로그램

개발 환경

HTML / CSS / JavaScript 크롬 확장프로그램 API

[4인 팀프로젝트]

소스코드

여러 OTT 콘텐츠를 각 플랫폼에서 접근하는게 불편하여 한 곳에서 접근하기 위해 개발했습니다.

크롬 확장프로그램을 설치하고 OTT 플랫폼 페이지에 접속하면, 콘텐츠의 우측 하단에 버튼이 생성되고 해당 버튼을 통해 관심 콘텐츠를 추가할 수 있도록 했습니다.

찜한 콘텐츠의 정보를 서버DB에 저장하고 불러올 수 있도록 했고, 팝업창에서 사용자의 콘텐츠 목록을 조회하도록 했습니다.

DB 서버는 NodeJS 로 구현했고, 사용자 화면은 HTML, CSS, JavaScript를 통해 구현했습니다.

로그인 기능을 구현하고, 비밀번호를 암호화해서 저장했으며 각 유저별로 찜한 목록을 만들기 위해 테이블을 설계하고 프론트엔드에서 각 유저별 정보를 불러와서 화면에 표현했습니다.

역할

API, DB 설계 및 크롬 확장 프로그램 구현

구현에 앞서 ERD 와 테이블 상세를 준비했고, 어떻게 테이블을 구성할지 고민했습니다.

기본 키를 잘 선택해야함을 느꼈고, 적절히 복합키를 선택했습니다. 인덱싱을 활용하여 자주 조회하는 컬럼에 대한 검색 속도를 높였습니다.

API를 설계하는 과정에서 어떤 종류의 데이터를 주고받아야 하는지 고민했고, 서버와 사용자 화면의 데이터 일관성 문제에 대해 고민했습니다.

외관상 버튼 하나를 추가하는 것은 단순해보이지만, 그 안에서 꽤 많은 종류의 데이터가 오가고 생각해보아야 할 점이 많다는 걸 느끼게 됐습니다.
